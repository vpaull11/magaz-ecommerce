# 🛒 Magaz — Современный интернет-магазин на Go

[![Go Report Card](https://goreportcard.com/badge/github.com/your-repo/magaz)](https://goreportcard.com/report/github.com/your-repo/magaz)
[![Go Version](https://img.shields.io/badge/Go-1.21%2B-blue.svg)](https://go.dev/)
[![Docker](https://img.shields.io/badge/Docker-Supported-2496ED.svg)](https://www.docker.com/)

**Magaz** — это полнофункциональная, производительная и SEO-оптимизированная платформа электронной коммерции, написанная на языке Go. Проект использует надёжный стек технологий для обеспечения высокой скорости работы, безопасности и удобного администрирования.

---

## 🌟 Ключевые возможности

### 🛍️ Для покупателей
- **Каталог товаров** с поддержкой вложенных и многоуровневых категорий.
- **Динамические атрибуты**: кастомные параметры и фильтры (например: "Объем памяти", "Цвет", "Вес").
- **Личный кабинет**: управление адресами доставки, списком желаний (Wishlist) и историей заказов.
- **Корзина и Checkout**: полноценный процесс оформления заказа.
- **Современный адаптивный дизайн**: интерфейс полностью оптимизирован для мобильных устройств (удобное отображение навигации, скрываемое боковое меню, фильтры по характеристикам и нижний бар) и генерируется динамически через Go templates по концепции "Всё на SSR".

### ⚙️ Для администраторов
- **Защищённая Админ-панель** (`/admin`).
- **Управление товарами и категориями**: загрузка фотографий, настройка атрибутов.
- **Статус заказов**: отслеживание и изменение статусов ("В обработке", "Отправлен" и т.д).
- **Управление глобальными настройками (SEO)**: редактирование robots.txt, Site Title, описания и OG Image напрямую из UI.

### 💳 Архитектура и Интеграции
- **Платежный микросервис**: Обработка платежей (Payment Gateway) полностью вынесена в независимый микросервис. Это гарантирует отказоустойчивость, безопасность платежных данных и позволяет масштабировать процесс оплаты независимо от ядра магазина.

### 🚀 SEO и Техническая оптимизация
- **ЧПУ (SEO-ссылки)**: URL-структура на основе пути (напр. `/catalog/smartphones` вместо `/?cat=smartphones`).
- **Декларативное SEO**: динамические теги OpenGraph, Twitter Cards, `canonical` URL.
- **Микроразметка**: встроенный JSON-LD Product schema.
- **Sitemap & Robots**: автоматическая динамическая отдача `/sitemap.xml` и `/robots.txt`.
- **Rate Limiters**: защита от спама и парсеров на глобальном уровне и уровне авторизации.

## 🛠 Технологический стек

* **Архитектура**: Основное ядро (Монолит) + **Изолированный микросервис платежей**.
* **Язык**: [Go (Golang)](https://go.dev/)
* **Роутер**: [go-chi/chi](https://github.com/go-chi/chi) (v5)
* **База данных**: PostgreSQL 
* **Работа с БД**: `database/sql`, расширение [jmoiron/sqlx](https://github.com/jmoiron/sqlx)
* **Сессии & CSRF**: `gorilla/sessions`, `gorilla/csrf`
* **Миграции**: Встроенный механизм миграций (на основе `.sql` файлов из папки `migrations`)
* **Деплой**: Docker & Docker Compose (поднятие основного приложения, БД и платежного шлюза)

## 📦 Установка и запуск (Docker)

Проект полностью контейнеризирован. Самый простой способ запустить его — использовать Docker Compose.

1. **Клонируйте репозиторий:**
   ```bash
   git clone https://github.com/ВАШ_АККАУНТ/magaz.git
   cd magaz
   ```

2. **Задайте файл окружения (опционально):**
   При необходимости переопределите параметры в `.env`. По умолчанию `docker-compose.yaml` уже содержит нужные fallback переменные.

3. **Запустите проект:**
   ```bash
   docker compose up -d --build
   ```

4. **Проверьте работу:**
   * Главная страница: `http://localhost:8080` (или порт, указанный в docker-compose)
   * Админ-панель: `http://localhost:8080/admin`
   
> **Важно**: Миграции базы данных накатываются на схему автоматически при старте приложения (пакет `cmd/server/main.go`).

## 🖥 Локальная разработка (без Docker)

Если вы хотите запустить проект локально для разработки:

1. Поднимите локальный PostgreSQL.
2. Создайте базу данных `magaz` (или любое другое имя).
3. Экспортируйте строку подключения к БД и сопутствующие переменные:
   ```bash
   export DATABASE_URL="postgres://postgres:password@localhost:5432/magaz?sslmode=disable"
   export SESSION_SECRET="your-super-secret-session-key" # Опционально
   ```
4. Запустите сервер:
   ```bash
   go run ./cmd/server
   ```

## 📂 Структура проекта

* `cmd/server/` — Точка входа в приложение (`main.go`).
* `internal/` — Скрытая от импорта кем-либо, основная логика приложения:
  * `handler/` — HTTP контроллеры (перехват GET/POST, рендер шаблонов).
  * `service/` — Бизнес-логика (товары, корзина, SEO, пользователи).
  * `repository/` — Слой работы с БД (SQL запросы).
  * `models/` — Структуры данных (entities).
  * `middleware/` — Фильтры запросов (авторизация, CSRF, rate limiters).
* `migrations/` — SQL-скрипты миграций (`*.up.sql` / `*.down.sql`).
* `web/templates/` — HTML шаблоны (Frontend).
* `uploads/` — Папка назначения для загружаемых картинок (генерируется).

## 🛡 Безопасность

* Полная защита форм от CSRF-атак.
* Пароли хэшируются через `bcrypt`.
* `HttpOnly` Secure cookies для управления сессиями.
* Встроенный rate-limiter ограничивает количество запросов в минуту от одного IP (защита модулей авторизации).
* SQL-inject защита (строгая типизация параметров, `$1, $2` плейсхолдеры).
